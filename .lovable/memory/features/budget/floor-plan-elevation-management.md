# Memory: features/budget/floor-plan-elevation-management
Updated: now

Gestión de Alzados (Individuales y de Coordenadas): 1. Alzados de Coordenadas: Etiquetas con guion y prefijo de nivel completo (ej. '1A-1A1', '1B-1C'). Incluye paredes perimetrales e interiores (cross-side). Clasifica marcadores por posición física (min/max cuadrícula) para detectar tramos. Secciones usan 'overlapStart' y 'fullWallLength' para posicionar huecos con precisión. Main corners (ABCD) reciben prefijo de nivel automáticamente desde los marcadores del usuario. isV1Main/isV2Main usa regex /^\d*[A-D]$/i para detectar esquinas principales con prefijo. 2. Cross-side: Usa TODOS los marcadores (main + custom) para generar pares, no solo los custom. El barrido (sweep) se restringe al tramo Y/X entre los dos marcadores del par, no al edificio completo. Se usa perimeterLabels Set para evitar duplicar alzados ya generados como perimetrales. 3. Paredes invisibles: Se incluyen como secciones en los alzados (no se filtran) para mantener la integridad del conjunto completo de elevaciones. 4. Interacción: Arrastre o flechas (1mm/5mm) para mover huecos; sincronización con Plano (positionX); 'Supr' elimina. 5. Visualización: Ventanas en azul, puertas en rosa pálido. 4 cotas por hueco. Líneas de cota por sección sobre la total. Regla en pantalla completa. 6. Edición: Selector inline por tarjeta. Medidas siempre en mm. 7. Pantalla completa: Container extra ancho (+160px, +120px) para evitar corte de cotas en la derecha/abajo.
